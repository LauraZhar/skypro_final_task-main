Обучающий портал компании

Некоторые удобные команды Git:

  1. git init - Используется для клонирования проекта как репозитория Git.
  2. git clone - Клонирует репозиторий в новую директорию.
  3. git config - Устанавливает значения конфигурации Git:
    • git config --local user.name "Name"
    • git config --local user.email "your_email@example.com"
    • git config --global user.name "Name"
    • git config --global user.email "your_email@example.com"
  4. git remote add origin [repository_url] - Добавляет или подключается к удаленному репозиторию.
  5. git remote -v - Показывает подключенные удаленные репозитории.
  6. git status - Показывает статус файлов в вашем локальном репозитории.
  7. git add - Переносит изменения в раздел добавленных файлов:
    • git add -A - Для добавления всех изменений.
    • git add [filename] - Для конкретного файла.
  8. git reset - Используется для отмены изменений.
  9. git commit - Фиксирует изменения в репозитории:
    • git commit -m "Text message" - Добавляет сообщение к коммиту.
    • git commit --amend - Изменяет предыдущий коммит.
  10. git push - Отправляет закоммиченные файлы в удаленный репозиторий:
    • git push -u origin [branch_name] - Устанавливает связь между локальной и удаленной ветками.
    • git push - Используется для отправки изменений в удаленный репозиторий.
  11. git fetch - Получает последнюю версию вашего локального репозитория.
  12. git pull - Извлекает и загружает изменения из удаленного репозитория.
  13. git branch - Создает, просматривает, переименовывает и удаляет ветки:
    • git branch -a - Предварительный просмотр всех веток в удаленном репозитории.
    • git branch -r - Предварительный просмотр всех веток на сервере.
  14. git checkout [branch_name] - Переключается на существующую ветку.
  15. git merge [branch_name] - Объединяет две ветки.
  16. git merge --abort - Отменяет слияние.
  17. git reset --hard HEAD - Удаляет все изменения и обновляет локальный репозиторий.
  18. git reset HEAD^ - Отменяет последний коммит.
  19. git diff - Сравнивает изменения.
  20. git rebase - Переписывает историю коммитов:
    • git rebase -i HEAD~N - Склеивает коммиты в интерактивном режиме.
  21. git stash - Сохраняет изменения для последующего использования.
  22. git stash pop - Применяет ранее отложенные изменения.
